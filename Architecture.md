### Data Flow Example (Create Record)

```
1. Frontend Request
   ┌──────────────────────────────────────────────────────────┐
   │ POST /api/records                                        │
   │ Headers: { Authorization: "Bearer <token>" }             │
   │ Body: { amount: 1000, type: "INCOME", ... }             │
   └──────────────────────────────────────────────────────────┘
                              │
                              ▼
2. withAuth() Middleware
   ┌──────────────────────────────────────────────────────────┐
   │ • Extract token from Authorization header                │
   │ • Verify JWT using JWT_SECRET                           │
   │ • Attach user { id, email, role } to request            │
   └──────────────────────────────────────────────────────────┘
                              │
                              ▼
3. Role Check
   ┌──────────────────────────────────────────────────────────┐
   │ • Check if user.role allows CREATE operation            │
   │ • ANALYST or ADMIN → Proceed                            │
   │ • VIEWER → Return 403 Forbidden                         │
   └──────────────────────────────────────────────────────────┘
                              │
                              ▼
4. Zod Validation
   ┌──────────────────────────────────────────────────────────┐
   │ • Validate request body against CreateRecordSchema      │
   │ • Check: amount > 0, type valid, category exists...     │
   │ • Invalid → Return 400 with validation errors           │
   └──────────────────────────────────────────────────────────┘
                              │
                              ▼
5. Database Operation (Prisma)
   ┌──────────────────────────────────────────────────────────┐
   │ • Create record in FinancialRecords table               │
   │ • Set userId = authenticated user.id                    │
   │ • Set createdAt, updatedAt timestamps                   │
   └──────────────────────────────────────────────────────────┘
                              │
                              ▼
6. Response
   ┌──────────────────────────────────────────────────────────┐
   │ 201 Created                                              │
   │ { status: 201, message: "Record created", data: {...} } │
   └──────────────────────────────────────────────────────────┘

   ```
